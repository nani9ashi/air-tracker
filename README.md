# QUUKI — 自転車の空気入れリマインダー

自転車のタイヤに空気を入れてからの経過日数を記録し、「あと○日」を画面の主役として見せるモバイル前提のアプリです。サイクルが近づくと色とメッセージが変わり、入れ忘れに気づけます。

ロードバイクのタイヤは2週間ほどで空気が抜けるのに、いつ入れたかを自分が覚えていられない——それを解決するために作り、自分でも毎日使っています。

**無料のPWA版を公開中**: https://nani9ashi.github.io/air-tracker/

## 使ってみる

1. 上のURLをスマホのブラウザで開く。
2. ブラウザのメニューから「ホーム画面に追加」を選ぶと、アプリとしてスタンドアロン起動できます。
3. 一度読み込めばオフラインでも動きます(Service Worker によるキャッシュ)。

データはすべて端末内(localStorage)に保存され、サーバには送信されません。Google Play 向けの Android 版(Capacitor)は準備中です。

## 主な機能

- **あと○日の表示** — 空気を入れてからの経過をリングで表示。残り日数に応じて色とメッセージが変化。
- **記録** — 「空気入れた！」ボタンから今日・昨日・任意の日付で記録。
- **推奨サイクル** — 7 / 14 / 21 / 28 日から選択。
- **履歴・統計** — 記録の一覧、平均間隔などの統計、ヒートマップ。
- **リマインド通知(基本)** — サイクル到来を知らせる通知。
- **テーマ** — ダーク / ライト(OS設定への自動追従つき)。
- **オフライン動作** — 通信なしで全機能が動作。

一部の機能(全履歴の閲覧・カスタム周期・バックアップ・複数台管理など)はロック表示になっています。準備中の Android 版で、アプリ内購入として開放する予定です。ロック中の機能も「何がそこにあるか」は見えるようにしています(データを消さず、鍵をかける方針)。

## 技術構成

| 項目 | 採用 |
|---|---|
| UI | React + Vite(プレーン JS) |
| 状態管理 | 自前の小さな store + localStorage(外部ライブラリなし) |
| 配信 | PWA(Service Worker + manifest)。GitHub Actions で GitHub Pages へ自動デプロイ |
| ネイティブ化 | Capacitor(Android・準備中)。`--mode capacitor` では Service Worker を外してビルド |
| 計測 | GoatCounter(cookie なし・個人情報なし。本番ビルドかつサイトコード設定時のみ送信) |
| テスト | Vitest + jsdom |

## 設計上の判断

作りながら決めたことのうち、あとから効いているものを3つ書いておきます。

1. **データは最初から「複数自転車 × 複数メンテ項目」の形で持つ。** 画面に出すのは空気入れだけでも、将来チェーン注油などを足すときに体験もデータも壊さないため。旧形式のデータは起動時に自動マイグレーションします。
2. **課金の資格(entitlement)は boolean でなく段(`plan: free / pro / premium`)で持つ。** 上位段の追加を「設定の変更」で済ませるため。
3. **無料版の履歴は削除ではなくロック/ぼかし表示。** ユーザーのデータを消さないこと、ロックの先に何があるか見えることを優先しました。

| 統計 (`stats.js`) | 同値分割 + 境界値分析 | 94 |

件数ではなく検出力で評価するため、意図的にバグ(ミュータント)を29体注入して測定しました。既存53件では28体中10体しか検出できませんでしたが、追加後は27体(96.4%)を検出します。テストで固定した現行挙動のうち8件は実装の疑義として記録し、到達経路と深刻度を独立に検証しています(いずれも未修正。製品ロジックは変更していません)。
