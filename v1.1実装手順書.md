# Air Tracker v1.1 実装手順書

v1（コア＋PWA）公開済み。v1.1 は **PWA のまま** 機能を足してエンゲージメントを高める。
v1 と同じ方針：**セマンティックトークンを使う / 小さく作る / フェーズ末で受け入れ基準を満たしコミット**。
スタックは据え置き（React+Vite・プレーンJS。TypeScript化は本バージョンでは行わない）。

- **方針の正**: `README.md`（§7 ロードマップ v1.1）
- **デザインの正**: `design-reference.css` ＋ 既存コンポーネント
- **ブランチ**: `feat/v1.1`

## スコープ（v1.1 でやること）
1. 履歴一覧 ＋ 統計（平均間隔・連続達成）
2. ヒートマップ（草風カレンダー）
3. 履歴の編集 / 削除
4. データのエクスポート / インポート（バックアップ）
5. 複数自転車の切替UI（追加・改名・削除・切替）

## フェーズ別ビルド手順

### P1.1-A — 履歴データ基盤 ＋ 統計ロジック（UIなし）
**目的**: 後続の全機能の土台。純粋関数とstore APIを先に固める。
作業:
1. `lib/stats.js`: 純粋関数を実装＋単体テスト。
   - `averageIntervalDays(history)`: 連続するpump間隔の平均（要素2未満は null）。
   - `currentStreak(history, intervalDays)`: 直近から連続して「間隔 ≤ intervalDays」で入れ続けた回数。
   - `sortedHistory(history)`: 日付昇順に正規化。
2. `store.js`: 履歴の編集・削除APIと、各履歴への安定 `id` 付与。
   - `pump` で `{ id, date }` を追記。`editHistory(id, dateISO)` / `removeHistory(id)`。
   - 編集・削除後は最新履歴から `lastReset` を再計算。
   - マイグレーションで既存 `{date}` 履歴に `id` を補完。
**受け入れ基準**: stats とstore のテストが全pass。既存データが壊れず `id` 補完される。

### P1.1-B — 履歴画面 ＋ ナビ拡張
**目的**: 履歴と統計を見られるようにする。
作業:
1. BottomNav を3タブ化（ホーム / 履歴 / 設定）。
2. `HistoryScreen`: 統計タイル（平均間隔・連続達成・記録数）＋ 履歴リスト（日付・曜日・前回からの間隔）。
3. 空状態（履歴なし）の表示。
**受け入れ基準**: pump すると履歴に追加され一覧に出る。統計が正しく表示。ライト/ダーク両対応。

### P1.1-C — ヒートマップ（草風）
**目的**: 継続を一目で。
作業:
1. `Heatmap` コンポーネント: 直近N週間の日別pump有無をマス目で表示（GitHub草風）。色は `--accent` 系の濃淡。
2. 履歴画面に組み込み。凡例＋月ラベル。色だけに頼らずtitle/aria。
**受け入れ基準**: pump 日にマスが着色。横スクロール/レスポンシブで破綻なし。両テーマ対応。

### P1.1-D — 履歴の編集 / 削除
**目的**: 記録を直せるように。
作業:
1. 履歴リスト各行から編集（日付変更）・削除。確認のうえ `editHistory` / `removeHistory`。
2. 変更が `lastReset`・カウントダウン・統計・ヒートマップに即反映。
**受け入れ基準**: 編集・削除が永続化され全表示に反映。最後の1件を消したら未記録状態に戻る。

### P1.1-E — バックアップ（エクスポート / インポート）
**目的**: データ移行・保全。
作業:
1. 設定画面: エクスポート（state を JSON ファイルでダウンロード）。
2. インポート（JSON ファイル選択 → 検証 → マイグレーション通して置換）。不正データは弾く。
**受け入れ基準**: エクスポート→インポートで状態が完全復元。壊れたファイルは安全に拒否。

### P1.1-F — 複数自転車の切替UI
**目的**: データ構造に既にある複数自転車を使えるように。
作業:
1. store: `addBike(name)` / `removeBike(id)` / `setActiveBike(id)` / 既存 `setBikeName`。
2. UI: ホーム/設定から自転車の切替・追加・改名・削除。activeBikeId で全画面が連動。
3. 各自転車が独立した air 項目（lastReset/interval/history）を持つ。
**受け入れ基準**: 自転車を追加・切替するとホーム/履歴/ヒートマップが対象自転車に切り替わる。削除時のactive再選択が安全。

### P1.1-G — テーマ切替UIの仕上げ ＋ ポリッシュ
**目的**: 仕上げ。
作業:
1. テーマ切替の体験改善（トランジション抑制でのちらつき回避、選択中の明示）。
2. 全画面のa11y/余白/モーション最終調整。空状態・エラーの文言整備。
**受け入れ基準**: 主要操作がキーボードのみで可能。両テーマで破綻なし。Lighthouse a11y 良好。
